TT;DR: Add the GUI companion to every existing installer/updater/uninstaller without changing normal non-Cargo CLI ownership behavior, and implement the explicitly approved two-step Cargo takeover.

## Why
Direct operator order. Current users must receive and remove the desktop app through the lifecycle they already trust.

## Plan
Assemble target-specific composite payloads; extend managed PowerShell/shell, four Windows native lanes, and universal macOS PKG; bundle private GTK runtimes for all Linux targets; add atomic staging, verification, rollback, repair, GUI shutdown, shortcuts/desktop registration, and complete uninstall; qualify the Cargo first-update/second-takeover exception.

## Impact
Touches the highest-risk release surface. Normal owner/channel/scope semantics must remain unchanged outside the documented Cargo exception.

## Acceptance
Every owner receives the GUI through update, repairs missing companions at the current version, rolls back atomically, and removes the complete product through `sd300 uninstall`.
