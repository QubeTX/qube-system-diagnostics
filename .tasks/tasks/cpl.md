TT;DR: Add the GUI companion to every existing installer/updater/uninstaller without changing normal non-Cargo CLI ownership behavior, and implement the explicitly approved two-step Cargo takeover.

## Why
Direct operator order. Current users must receive and remove the desktop app through the lifecycle they already trust.

## Plan
Assemble target-specific composite payloads; extend managed PowerShell/shell, four Windows native lanes, and universal macOS PKG; bundle private GTK runtimes for all Linux targets; add atomic staging, verification, rollback, repair, GUI shutdown, shortcuts/desktop registration, and complete uninstall; qualify the Cargo first-update/second-takeover exception.

## Impact
Touches the highest-risk release surface. Normal owner/channel/scope semantics must remain unchanged outside the documented Cargo exception.

## Acceptance
Every owner receives the GUI through update, repairs missing companions at the current version, rolls back atomically, and removes the complete product through `sd300 uninstall`.

## Verification
- [ ] Synthetic-prior update/repair/uninstall matrices pass for every owner
- [ ] All target payloads are checksum-pinned, provenance-attested, path-leak-free, and license-complete
- [ ] Blank-host Linux and signed/notarized macOS package gates pass

## Status
Active. The GUI companion is wired into both WiX MSI lanes, both Inno EXE lanes, the managed PowerShell installer, the managed macOS/Linux shell installer, and the universal PKG construction path. The updater now stages and verifies companion payloads, requests graceful GUI shutdown, distinguishes complete-current no-op from same-version repair, and tracks rollback material; uninstall removes owned GUI integrations while preserving user exports. A physical Corporate MSI vertical slice passed. The macOS workflow now builds both native architectures and assembles a universal signed/notarized app, while a new Linux workflow packages pinned private GTK closures for GNU x86-64/ARM64 and musl x86-64. Every managed installer now verifies the outer checksum plus exact internal manifest identity, inventory, and file hashes before live mutation. Cargo-dist's generic CLI-only MSI producer has been removed; the legacy MSI URL is now an exact byte alias of the qualified composite Global MSI. Linux packaging now traces every copied runtime library, module, and schema to its Ubuntu or Alpine owner package, bundles package copyright/common-license evidence, and emits package-to-file SPDX relationships; hosted GNU and musl builds must still prove the metadata and doc-subpackage closure. Hosted builds, all-owner synthetic-prior execution, Inno qualification, Global MSI qualification, Cargo two-step takeover, rollback/failure injection, and Makira embedding-license evidence remain release-blocking.
